Apple Music User Token Helper local setup

Use this page to sign in with Apple Music in your browser and extract the Music User Token needed by this pi extension.

What this page helps you do

This project is currently local-only. You bring your own Apple Music API credentials, then this page helps you finish the browser sign-in flow and copy the final musicUserToken.

  1. Create or locate your Apple Music / MusicKit setup in the Apple Developer portal.
  2. Create a MusicKit private key and download the .p8 file.
  3. Copy your Team ID and Key ID.
  4. Generate a developer token with the script in this repo.
  5. Paste that developer token into this page.
  6. Click Configure MusicKit, then Sign in & get user token.
  7. Copy the generated config and save it to .pi/apple-music.json.

Before you start

  1. You need an Apple Developer account.
  2. You need an Apple account with an active Apple Music subscription.
  3. If your Apple Music web configuration requires allowed origins, add http://localhost:8787.
  4. If browser sign-in is blocked, try Safari and allow popups.

Apple Developer setup checklist

  1. Open the Apple Developer portal.
  2. Create or choose the app / identifier you want to use with MusicKit.
  3. Enable Apple Music / MusicKit for that app configuration if required in your setup.
  4. Add http://localhost:8787 as an allowed origin if needed.
  5. Create a MusicKit key.
  6. Copy the Key ID.
  7. Copy your Team ID.
  8. Download the private key file (.p8) and keep it safe.

Generate your developer token

Run this in the repo before using the form below:

npm run generate-developer-token -- \
  --team-id <APPLE_TEAM_ID> \
  --key-id <APPLE_KEY_ID> \
  --private-key <PATH_TO_AUTHKEY_P8>

Config snippet

{
  "developerToken": "",
  "musicUserToken": "",
  "storefront": "us"
}

Next step

Save the token into .pi/apple-music.json:

mkdir -p .pi
cat > .pi/apple-music.json <<'EOF'
{
  "developerToken": "PASTE_DEVELOPER_TOKEN_HERE",
  "musicUserToken": "PASTE_MUSIC_USER_TOKEN_HERE",
  "storefront": "us"
}
EOF

You can also save the same config in ~/.pi/agent/apple-music.json if you want it available across projects.
